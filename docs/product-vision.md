# Product Vision

## 一句话定义

基于 X API 的个人书签管理工具，自动抓取并智能归类用户的 X 书签内容。

## 核心问题

X (Twitter) 的书签功能没有分类能力，书签数量增长后难以检索和管理，用户无法高效回顾和组织收藏的内容。

## 目标用户

有大量 X 书签、需要系统化管理和分类这些内容的个人用户（本地工具，单用户）。

## 不做什么（边界）

- 不做多用户 SaaS 平台
- 不做社交功能（分享、协作）
- 不做 X 以外平台的书签管理
- 不做内容创作或发布功能
- 当前阶段不接入 AI API 分类（后续由 OpenClaw 接管）

## 差异化原则

- **本地优先**：数据存储在本地 SQLite，不依赖外部数据库服务
- **增量同步**：只拉取新增书签，最小化 API 调用成本
- **可扩展分类**：关键词规则引擎为当前方案，架构预留 AI 分类接口（OpenClaw）
- **零门槛部署**：Next.js 一体化，`npm run dev` 即可使用

## 成功标准

- 能通过 OAuth 2.0 成功拉取用户全部书签
- 书签自动按关键词规则分类，准确率可接受
- 用户可手动调整分类
- 增量同步有效减少 API 调用次数
- 重启后本地数据不丢失
